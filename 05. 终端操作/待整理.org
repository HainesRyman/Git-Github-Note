
版本库区
    版本号：
        生成：
            用 SHA1 计算出来的具有很多位的数字
        格式：
            用十六进制表示
    版本的标识符：
        ~~HEAD~~ 倒数第 1 次提交的版本（当前版本）（最近一次提交的版本）
        ~~HEAD^~~ 倒数第 2 次提交的版本
        ~~HEAD^^~~ 倒数第 3 次提交的版本
        ~~HEAD~`Number_N`~~ 倒数第 N + 1 次提交的版本
            ~HEAD~100~~ 倒数第 101 次提交的版本

回退：
    ~~git reset --hard HEAD^~~ 回退到倒数第 2 次提交的版本
        执行此命令后，如果想要取消此次回退，需要在同一个命令行session中执行~~git reset --hard `倒数第 1 次提交的版本号`~~
    ~~git reset --hard `某版本号`~~ 回退到某版本号对应的版本
        为了方便，可以省略输入版本号的全部数字，只输入版本号开头的任意长度的唯一的一串数字即可， Git 会自动匹配对应的版本号
回退的原理：
    Git 内部有一个指向当前版本的指针 HEAD
    当回退版本的时候， Git 仅仅把指针 HEAD 改变为指向其他版本，然后把工作区的文件更新
    所以回退版本的速度非常快

~~git reflog~~ 查看历次命令的执行情况，包括：版本号、动作、对象
